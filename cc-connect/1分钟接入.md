# 1 分钟接入：个人微信 × 本地 Agent（cc-connect）

> 把手机微信聊天桥接到本机 AI 编码 Agent（默认 Claude Code）。
> 两个版本：**For Human** 给人照着做；**For Agent** 喂给 AI Agent，让它根据你的系统环境自动帮你接入。

---

# For Human

**唯一前置**：本机已装好目标 Agent 的 CLI 且在 PATH（如 `claude` / `codex` / `opencode`）。

通用三步：**① 下载二进制 → ② 写最小 config.toml → ③ 扫码绑定并启动**。

## macOS

```bash
# ① 下载：到 https://github.com/chenhg5/cc-connect/releases/ 选合适版本下载
#    Apple Silicon → cc-connect-vX.Y.Z-darwin-arm64.tar.gz；Intel → darwin-amd64
#    解压并重命名（版本号换成你下的那个）：
tar xzf cc-connect-vX.Y.Z-darwin-arm64.tar.gz && mv cc-connect-vX.Y.Z-darwin-arm64 cc-connect
xattr -d com.apple.quarantine cc-connect 2>/dev/null  # 去 Gatekeeper 隔离标记

# ② 写最小配置（见下方 config.toml）

# ③ 扫码绑定 + 启动
./cc-connect weixin setup --project my-wechat   # 手机微信扫终端二维码 → 确认登录 → token 自动写回 config.toml
./cc-connect                                     # 启动服务
```

## Windows（PowerShell）

```powershell
# ① 下载：到 https://github.com/chenhg5/cc-connect/releases/ 选合适版本下载
#    Windows x64 → cc-connect-vX.Y.Z-windows-amd64.exe；下载后重命名为 cc-connect.exe（方便下面命令）

# ② 写最小配置（见下方 config.toml）

# ③ 扫码绑定 + 启动
.\cc-connect.exe weixin setup --project my-wechat   # 手机微信扫终端二维码 → 确认登录 → token 自动写回 config.toml
.\cc-connect.exe                                     # 启动服务
```

## 最小 config.toml（两个系统通用）

```toml
[log]
level = "info"

[[projects]]
name = "my-wechat"

[projects.agent]
type = "claudecode"            # 换别的 agent 改这里：codex / opencode / gemini ...

[projects.agent.options]
work_dir = "/path/to/your/project"   # Agent 工作目录（Windows 用 C:\path\to\project）

[[projects.platforms]]
type = "weixin"
# token / account_id 扫码后自动写入下面
```

## 完成

从手机微信给扫码后出现的那个 bot 发任意一条消息（如「你好」），收到回复即接入成功。验证日志：
```
weixin: ilink ready-for-poll          # ilink 连通
platform ready                         # 平台就绪
message received  user=xxx@im.wechat   # 收到消息
session spawned  → turn complete       # 本地 Agent 被拉起并回复
```

# For Agent

你是一个工程助手。用户想通过 **cc-connect** 把「个人微信」接到本机的 AI 编码 Agent（默认 Claude Code；也可能是 codex / opencode / gemini 等，按用户说的来）。
你的任务：根据用户当前机器的操作系统与环境，把接入跑通。先探测再行动，关键决策点向用户确认。

## 0. 探测环境
- 操作系统/架构：`uname -s` 与 `uname -m`（Windows PowerShell 用 `$env:OS` / `[Environment]::Is64BitOperatingSystem`）。
  - Darwin + arm64 → `darwin-arm64`；Darwin + x86_64 → `darwin-amd64`
  - Windows + x64 → `windows-amd64.exe`；Windows + arm64 → `windows-arm64.exe`
- 目标 Agent CLI 是否在 PATH：macOS/Linux `command -v claude`；Windows `Get-Command claude`。没装就先让用户装好再来。
- 若用户想从源码编译：需要 Go 1.25+（`go version`）；不满足就**用预编译二进制**（推荐，省事）。

## 1. 获取 cc-connect
到 https://github.com/chenhg5/cc-connect/releases/ 取对应 OS/arch 的产物：
- macOS：`.tar.gz` → `tar xzf` 解压、重命名为 `cc-connect`、`xattr -d com.apple.quarantine cc-connect` 去隔离标记。
- Windows：`.exe` → 重命名为 `cc-connect.exe`。
验证可运行：`./cc-connect --version`（Windows `.\cc-connect.exe --version`）。

## 2. 写最小 config.toml（cc-connect 默认读当前目录的 config.toml）
向用户要两个值（或用合理默认并告知）：
- 项目名（如 `my-wechat`）
- Agent 工作目录 `work_dir`（Agent 在哪个项目里干活）

然后写入：
```toml
[log]
level = "info"

[[projects]]
name = "<项目名>"

[projects.agent]
type = "claudecode"            # 换别的 agent 改这里

[projects.agent.options]
work_dir = "<工作目录>"

[[projects.platforms]]
type = "weixin"
```
用 `cc-connect --config ./config.toml config format` 校验能否解析。

## 3. 扫码绑定（这一步必须真人扫码）
运行 `cc-connect weixin setup --project <项目名>`，它会在终端打印二维码并等待。
⚠️ **你（Agent）无法扫码**——明确告诉用户：用手机微信扫终端里的二维码，并在手机上点「确认登录」。
成功标志：终端打印 `✅ 已与微信建立连接`，并自动把 `token` / `base_url` / `account_id` 写回 config.toml。

## 4. 启动 + 验证
- 启动 cc-connect（先前台跑便于看日志）：`./cc-connect`（Windows `.\cc-connect.exe`）。
- 让用户从微信给扫码后出现的 bot 发一条消息（如「你好」）。
- 日志依次出现即成功：
  ```
  weixin: ilink ready-for-poll
  platform ready
  message received  user=xxx@im.wechat
  session spawned  → turn complete
  ```

## 5. 务必提醒用户（安全与风险）
- **设 `allow_from`**：默认不设 = 任何人都能驱动本机 Agent。从日志 `user=...@im.wechat` 取到用户 ID 后写进 `[projects.platforms.options] allow_from = "..."`，重启生效。
- **ilink 是第三方网关**（非微信官方），有封号风险，仅自用、别用主力号高频。
- 想常驻/开机自启：`cc-connect daemon install`（macOS=launchd，Windows=schtasks）。
- 想要更合规的微信通道：改用企业微信（`type = "wecom"`）。

## 边界
- 扫码、手机确认登录是**真人动作**，你只能引导、不能代劳。
- ilink 报错（账号未开通等）是网关侧限制，非配置问题。
- 深入文档（配置详解 / 原理 / Prompt 组成 / 注意事项）见同目录 `微信对接笔记.md`。

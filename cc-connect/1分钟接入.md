# 1 分钟接入：个人微信 × 本地 Agent（cc-connect）

> 把手机微信聊天桥接到本机 AI 编码 Agent（默认 Claude Code）。
> 两个版本：**For Human** 给人照着做；**For Agent** 喂给 AI Agent，让它根据你的系统环境自动帮你接入。

---

# For Human

**唯一前置**：本机已装好目标 Agent 的 CLI 且在 PATH（如 `claude` / `codex` / `opencode`）。

通用三步：**① 下载二进制 → ② 写最小 config.toml → ③ 扫码绑定并启动**。

> ⚠️ **贯穿全流程的一条规则**：子命令必须紧跟在 `cc-connect` 后面、作为**第一个参数**。`--config`、`--project` 等选项一律放在子命令**之后**。写反（如 `--config … weixin setup`）会被当成启动守护进程，而不是执行子命令。

## macOS

```bash
# ① 下载：到 https://github.com/chenhg5/cc-connect/releases/ 选合适版本下载
#    Apple Silicon → cc-connect-vX.Y.Z-darwin-arm64.tar.gz；Intel → darwin-amd64
#    解压并重命名（版本号换成你下的那个）：
tar xzf cc-connect-vX.Y.Z-darwin-arm64.tar.gz && mv cc-connect-vX.Y.Z-darwin-arm64 cc-connect
xattr -d com.apple.quarantine cc-connect 2>/dev/null  # 去 Gatekeeper 隔离标记

# ② 写最小配置（见下方 config.toml）

# ③ 扫码绑定 + 启动
./cc-connect weixin setup --project my-wechat --set-allow-from-empty
#   ↑子命令在最前。--set-allow-from-empty：扫码即把你本人写进 allow_from（只有你能驱动 Agent）
#   手机微信扫终端二维码 → 确认登录 → token 自动写回 config.toml
#   终端二维码扫不出来时加 --qr-image login-qr.png，改扫图片
./cc-connect                                     # 启动服务
```

## Windows（PowerShell）

```powershell
# ① 下载：到 https://github.com/chenhg5/cc-connect/releases/ 选合适版本下载
#    Windows x64 → cc-connect-vX.Y.Z-windows-amd64.exe；下载后重命名为 cc-connect.exe（方便下面命令）

# ② 写最小配置（见下方 config.toml）

# ③ 扫码绑定 + 启动
.\cc-connect.exe weixin setup --project my-wechat --set-allow-from-empty
.\cc-connect.exe                                     # 启动服务
```

## 下载加速（直连 GitHub 困难时，国内常见）

若终端/浏览器访问 github.com 很慢或失败，任选一条：

- **代理前缀**（推荐）：在 release 下载链接前拼 `https://gh-proxy.com/`。完整链接形如（粘到浏览器或下载工具，换版本号/平台即可）：
  ```
  https://gh-proxy.com/https://github.com/chenhg5/cc-connect/releases/download/v1.3.4/cc-connect-v1.3.4-darwin-arm64.tar.gz
  ```
  若 `gh-proxy.com` 偶尔不通，再换同类服务：`https://mirror.ghproxy.com/`、`https://ghps.cc/` 等（这类公共服务会轮换，挑当前能用的）。
- **浏览器加速站**：用 GitHub 加速镜像站打开 releases 页面，直接在浏览器下载（适合终端下不动的情况）。
- **下不到就编译**：本机有 Go 1.25+ 时，`git clone` 源码后 `make build`（源码同样可走代理/镜像拉取）。注意本机 Go 太旧（如 1.16）编不了，需先升级。

## 最小 config.toml（两个系统通用）

```toml
[log]
level = "info"

[[projects]]
name = "my-wechat"

[projects.agent]
type = "claudecode"            # 换别的 agent 改这里：codex / opencode / gemini ...

[projects.agent.options]
work_dir = "/path/to/your/project"   # Agent 工作目录。Windows 用正斜杠最稳：E:/workspace/proj
```

> 说明：`weixin` 平台块**不需要手写**——`weixin setup` 会自动把它挂到项目上并把 token 写回。手写空的 `[[projects.platforms]] type = "weixin"` 反而会让某些校验报 `token is required`，没必要。

## 完成

从手机微信给扫码后出现的那个 bot 发任意一条消息（如「你好」），收到回复即接入成功。
> ⚠️ **每次启动/重启 cc-connect 后，都要发一条新消息**——早于进程启动时间的消息会被当旧消息丢弃。

验证日志：
```
weixin: ilink ready-for-poll          # ilink 连通
platform ready                         # 平台就绪
message received  user=xxx@im.wechat   # 收到消息
session spawned  → turn complete       # 本地 Agent 被拉起并回复
```

# For Agent

你是一个工程助手。用户想通过 **cc-connect** 把「个人微信」接到本机的 AI 编码 Agent（默认 Claude Code；也可能是 codex / opencode / gemini 等，按用户说的来）。
你的任务：根据用户当前机器的操作系统与环境，把接入跑通。先探测再行动，关键决策点向用户确认。

> ⚠️ **最高频踩坑：子命令必须在第一个参数。** cc-connect 按 `os.Args[1]` 派发：`weixin setup --project X --config ./config.toml` ✅；一旦写成 `--config … weixin setup`，`os.Args[1]` 是 `--config`，匹配不到任何子命令 → **会去跑默认的守护进程**，于是你看到的是"启动服务"的日志而非 setup。任何子命令（`weixin` / `config` / `feishu` / `daemon` …）都遵循此规则，`--config` 等选项只能放子命令**之后**。

## 0. 探测环境
- 操作系统/架构：`uname -s` 与 `uname -m`（Windows PowerShell 用 `$env:OS` / `[Environment]::Is64BitOperatingSystem`）。
  - Darwin + arm64 → `darwin-arm64`；Darwin + x86_64 → `darwin-amd64`
  - Windows + x64 → `windows-amd64.exe`；Windows + arm64 → `windows-arm64.exe`
- 目标 Agent CLI 是否在 PATH：macOS/Linux `command -v claude`；Windows `Get-Command claude`。没装就先让用户装好再来。
- 若用户想从源码编译：需要 Go 1.25+（`go version`）；不满足就**用预编译二进制**（推荐，省事）。

## 1. 获取 cc-connect
到 https://github.com/chenhg5/cc-connect/releases/ 取对应 OS/arch 的产物：
- macOS：`.tar.gz` → `tar xzf` 解压、重命名为 `cc-connect`、`xattr -d com.apple.quarantine cc-connect` 去隔离标记。
- Windows：`.exe` → 重命名为 `cc-connect.exe`。
- ⚠️ **若直连 GitHub 失败/很慢（国内常见，表现为下载只有几十 KB/s 甚至超时）**：直接用代理前缀 `https://gh-proxy.com/`（不要先硬刚直连浪费时间），或让用户用浏览器加速站下好后放到当前目录；本机有 Go 1.25+ 也可 `git clone` 源码后 `make build`。
验证可运行：`./cc-connect --version`（Windows `.\cc-connect.exe --version`）。

## 2. 写最小 config.toml（cc-connect 默认读当前目录的 config.toml）
向用户要两个值（或用合理默认并告知）：
- 项目名（如 `my-wechat`）
- Agent 工作目录 `work_dir`（Agent 在哪个项目里干活）

然后写入：
```toml
[log]
level = "info"

[[projects]]
name = "<项目名>"

[projects.agent]
type = "claudecode"            # 换别的 agent 改这里

[projects.agent.options]
work_dir = "<工作目录>"         # Windows 用正斜杠，如 E:/workspace/proj；反斜杠需 \\ 转义
```

> **不要手写 `[[projects.platforms]]` 平台块，也不要在这一步做 config 校验。** 原因：(1) `weixin setup` 会自动挂载 weixin 平台并把 token 写回；(2) 此时还没有 token，任何"构造平台/严格校验"类的命令都会报 `token is required`，造成"配置坏了"的误判。真正的校验是：步骤 3 的 setup 成功 + 步骤 4 的 daemon 正常起。

## 3. 扫码绑定（这一步必须真人扫码）
运行（**子命令在前**，推荐两个 flag）：
```bash
./cc-connect weixin setup --project <项目名> --set-allow-from-empty --qr-image login-qr.png
```
- `--set-allow-from-empty`：扫码成功即把扫码人的微信 ID 写进 `allow_from`，**当场锁权限**（只有 ta 能驱动 Agent）。比"先开放、事后从日志抄 ID 再加锁"更安全，省一步。
- `--qr-image <path>`：把二维码存成 PNG。终端字符画二维码经常扫不出来，让用户改扫这张图更稳。
- ⚠️ **你（Agent）无法扫码**——明确告诉用户：用手机微信扫二维码（终端里的或 PNG），并在手机上点「确认登录」。二维码约 110 秒过期，setup 会自动刷新最多 3 次。
- 成功标志：终端打印 `✅ 已与微信建立连接`，并自动把 `token` / `base_url` / `account_id` / `allow_from` 写回 config.toml。

## 4. 启动 + 验证
- 启动 cc-connect（先前台跑便于看日志）：`./cc-connect`（Windows `.\cc-connect.exe`）。
- ⚠️ **启动/重启后，让用户发一条"新"消息**（如「你好」）。**早于进程启动时间的旧消息会被当旧消息静默丢弃**（防重放）——这是"发了消息却没反应"的最常见原因，要么重启后发新消息，要么排查时开 debug（见步骤 6）。
- 日志依次出现即成功：
  ```
  weixin: ilink ready-for-poll
  platform ready
  message received  user=xxx@im.wechat
  session spawned  → turn complete
  ```

## 5. 务必提醒用户（安全与风险）
- **`allow_from`**：若步骤 3 用了 `--set-allow-from-empty`，已自动锁为扫码人；否则默认不设 = 任何人都能驱动本机 Agent，需从日志 `user=...@im.wechat` 取 ID 写进 `[projects.platforms.options] allow_from = "..."` 后重启。
- **ilink 是第三方网关**（非微信官方），有封号风险，仅自用、别用主力号高频。
- 想常驻/开机自启：`./cc-connect daemon install`（macOS=launchd；Windows `.\cc-connect.exe daemon install`=schtasks）。
- 想要更合规的微信通道：改用企业微信（`type = "wecom"`）。

## 6. 排错（收不到消息 / 无反应时）
- **先把 `[log] level` 改成 `debug` 重启**：以下两类丢弃在 `info` 级是**静默**的，只有 `debug` 才打日志——
  - `weixin: sender not in allow_from from=...` → 发送者不在白名单（换号发了，或 allow_from 没配对）。
  - `weixin: skip old message time=...` → 消息早于进程启动时间，被防重放丢弃；**重启后重新发一条新消息**即可。
- 子命令写错顺序（`--config` 放最前）会静默变成"启动守护进程"，表现为 setup 没打印二维码却出现 `acquired instance lock / config loaded`——核对命令顺序。
- 长轮询空闲时日志**不会**输出（正常），别误判为"卡住"；只有收到消息才打 `message received`。

## 边界
- 扫码、手机确认登录是**真人动作**，你只能引导、不能代劳。
- ilink 报错（账号未开通等）是网关侧限制，非配置问题。

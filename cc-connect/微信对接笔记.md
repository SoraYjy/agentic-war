# cc-connect × 个人微信 × 本地 Agent —— 接入指南

> 通过 **cc-connect** 把「手机微信聊天」桥接到本机的 AI 编码 Agent（Claude Code / Codex / OpenCode 等）。在微信里发消息，就能远程驱动本机 Agent 干活（读改代码、跑命令）。
>
> 本指南覆盖 **macOS 与 Windows** 的个人微信（ilink 通道）接入。
>
> ⏱️ 只想快速跑通？看 [1分钟接入.md](1分钟接入.md)（含可喂给 AI Agent 的自动接入版）。

## 1. 配置文件 config.toml 介绍

cc-connect 默认读取当前目录 `config.toml`（或 `--config <path>` 指定）。结构：**全局设置 + 多个项目**，每个项目把「一个工作目录 + 一个 Agent + 若干平台」绑在一起。

```toml
[log]
level = "info"            # info / debug（调试长轮询细节用 debug）

[display]
thinking_messages = true  # 是否把模型"思考过程"作为 💭 预览发到微信（不想要改 false）

[[projects]]
name = "my-wechat"        # 项目名；扫码、多项目管理时用它区分

[projects.agent]
type = "claudecode"       # Agent 类型（见下方支持列表）

[projects.agent.options]
work_dir = "/path/to/your/project"        # Agent 在哪个目录干活
mode = "default"          # 权限模式，见下表
model = "sonnet"          # 可选：指定模型（也可微信里 /model 切）

[projects.agent.options.env]
# 可选：给 Agent 进程注入环境变量（如沙箱隔离、第三方模型 API 等）
# CLAUDE_CONFIG_DIR = "<sandbox_dir>"
# ANTHROPIC_BASE_URL = "https://your-anthropic-compatible-api"

[[projects.platforms]]
type = "weixin"           # 平台类型

[projects.platforms.options]
token = "扫码自动写入"               # ilink bot Bearer token（扫码回填）
base_url = "https://ilinkai.weixin.qq.com"
account_id = "扫码自动写入"          # 多账号时隔离本地状态
allow_from = "你的@im.wechat用户ID"  # ⚠️ 强烈建议设，限制谁能用（/whoami 取自己的 ID）
# chat_id = "xxx@chatroom"          # 可选：绑到某个群（群 ID 以 @chatroom 结尾）
```

**校验配置**（不启动服务，只看能否解析）：`./cc-connect --config ./config.toml config format`

### 权限模式 `mode`

| 值 | 行为 |
|----|------|
| `default` | 每次工具调用都要你在微信里回「允许」/「allow」 |
| `acceptEdits` | 文件编辑自动通过，其余仍问 |
| `auto` | Agent 自动判断，基本不弹窗（最省事） |
| `plan` | 只规划不执行，审批后再做 |
| `bypassPermissions` | 全部自动通过（=yolo，慎用） |
| `dontAsk` | 未预授权的工具自动拒绝（安全推荐） |

### 常用 Agent 配置项（`[projects.agent.options]`）

| 配置键 | 作用 |
|--------|------|
| `work_dir` | Agent 工作目录（也可微信里 `/dir` 临时切） |
| `mode` | 权限模式（见上表） |
| `model` | 模型（→ `--model`） |
| `cli_path` | Agent CLI 路径，默认 `claude` |
| `env` | 给 Agent 进程注入环境变量（子表） |
| `allowed_tools` / `disallowed_tools` | 预授权 / 禁用工具（数组） |
| `system_prompt` | 替换 Agent 默认系统提示 |
| `append_system_prompt` | 在默认系统提示后追加 |

### 支持的 Agent / 平台

- **Agent**：`claudecode` `codex` `cursor` `gemini` `copilot` `opencode` `qoder` `kimi` `iflow` `devin` `antigravity` `acp` `pi` `tmux`（换 agent 只改 `type`，前提是本机有对应 CLI）
- **平台**：个人微信(`weixin`)、企业微信、飞书、Telegram、Slack、Discord、钉钉、QQ、LINE 等（本指南聚焦个人微信）

## 2. 原理：cc-connect 如何在微信里和 Agent 交互

### 整体架构

```
手机微信 ── 微信服务器 ── [ilink 网关 ilinkai.weixin.qq.com] ──HTTP 长轮询──> cc-connect(本机) ──stream-json──> 本地 Agent CLI
                                                                                        (work_dir = 你的项目)
```

- **ilink 网关**是**第三方服务**（非微信官方），把个人微信协议翻译成 HTTP API。cc-connect 用 `getUpdates`（长轮询）收消息、`sendMessage` 发消息。
- **不需要公网 IP**：是 cc-connect 主动去轮询 ilink，不是 ilink 回调你。
- **本地 Agent**：cc-connect 用 `exec.LookPath` 找到本机 Agent CLI（如 `claude`），以 stream-json 方式交互。

### "微信 ClawBot" 是什么

扫码后微信里出现的「微信 ClawBot」**不是 Agent**，而是 **ilink 网关在微信侧给你开的 bot 联系人**（收发消息的入口，名字由网关侧定，"Claw" 源自上游项目 OpenClaw）。Agent 本身没有微信能力——靠 cc-connect 把这个入口和 Agent 接到一起：

```
你(微信) ──> 【微信 ClawBot】(ilink bot 联系人)
                  │ 消息进入 ilink 网关
                  ▼
            cc-connect（长轮询 getUpdates 把消息捞出来）
                  │ Send → Agent stdin (stream-json)
                  ▼
            本地 Agent（真正干活的大脑）
                  │ 回复
                  ▼
            cc-connect（sendMessage）──> ilink ──> ClawBot ──> 你的微信
```

### cc-connect 注入的指令

cc-connect 启动 Agent 时会追加一段系统提示（详见第 3 节），告诉 Agent：你在给一个 IM 平台（微信）当桥、回复用纯文本、别用 markdown 表格、用短段落等。所以 Agent "知道"自己在微信里——这是 cc-connect 注入的，不是泄漏。

### 会话生命周期

- 第一条消息后，**会话会一直保持**，不主动断开；后续消息复用同一会话、同一上下文。
- cc-connect / Agent 重启后会用 `--resume` 恢复原会话（除非 `/new`）。
- 管理会话：`/new` 重开、`/list` 查看、`/switch` 切换。

### 微信里常用的斜杠命令

| 命令 | 作用 |
|------|------|
| `/new` | 新建会话（清空上下文） |
| `/model <name>` | 切模型 |
| `/mode <name>` | 改权限模式 |
| `/dir <path>` | 切换 Agent 工作目录 |
| `/status` `/whoami` | 查状态、取自己的 User ID |

## 3. Prompt 的组成形式

发往模型的请求里有**三个互相独立的槽位**，各走各的道，**不互相拼接**：

| 槽位 | 装什么 | 怎么进去 | 在模型 API 里的位置 |
|------|--------|---------|-------------------|
| **① 系统提示** | cc-connect 注入的指令 + Agent 原生提示 + CLAUDE.md 记忆 | Agent CLI 的 `--append-system-prompt-file` / `--system-prompt` | API 的 `system` 字段 |
| **② 对话** | 你来我往的 user/assistant 轮次 | cc-connect 经 stdin 写 stream-json | API 的 `messages` 数组 |
| **③ 记忆** | `CLAUDE.md`（项目 + 全局） | Agent 自己加载，并入 ① | 进 `system` 上下文 |

### cc-connect 如何注入系统提示

Agent CLI（以 Claude Code 为例）有控制系统提示的标志：
- `--append-system-prompt-file <file>`：把文件内容**追加**到 Agent 原生系统提示之后（cc-connect 默认走这条）
- `--system-prompt "<text>"`：**整段替换** Agent 原生系统提示（config 设了 `system_prompt` 才用）

追加的文件由三层拼成：
1. cc-connect 的 `AgentSystemPrompt()`（"You are running inside cc-connect, a bridge…"）
2. ＋ 当前平台的格式提示（微信：`plain text to Weixin, avoid markdown tables, use short paragraphs`）
3. ＋ config 里的 `append_system_prompt`（可选）

→ 全进 ① 系统提示，**会话启动时设定一次**，之后每轮对话都带着。

### 你的消息如何进对话

每条微信消息 → cc-connect 的 `Send()` → 写进 Agent 的 stdin：
```json
{"type":"user","message":{"role":"user","content":"你在微信发的话"}}
```
这是对话里的一个 user 轮次，进 ② `messages`。

### 数据流

```
[会话启动, 一次] cc-connect --append-system-prompt-file <file> ──> Agent CLI
                                                                        │
[每条消息]     微信 ──> cc-connect ──Send()──> [stdin stream-json] ──> │
                                                                        ▼
                                          Agent 拼装后调用模型 API:
                                          ┌──────────────────────────────────┐
                                          │ system  = Agent 原生系统提示      │
                                          │        + cc-connect 追加内容 (①) │
                                          │        + CLAUDE.md 记忆 (③)      │
                                          │ messages= [{user: 你的话}, …] (②)│
                                          └──────────────────────────────────┘
                                                                        │
[回复]        微信 <── cc-connect 解析(text/thinking/tool) <── stream-json │
```

### 一句话总结
- **你发的** = 对话里的 user 消息（`messages`，走 stdin）
- **cc-connect 塞的** = 系统提示（`system`，走 `--append-system-prompt-file`）
- **CLAUDE.md** = Agent 自己的记忆（并入 `system`，由 Agent 加载）

## 4. 注意事项

### 🔴 安全
- **务必设 `allow_from`**：默认不设 = 任何人（尤其被拉进群后）都能驱动你本机 Agent。自己的 ID 用 `/whoami` 或看日志 `user=...@im.wechat` 取。
- **特权命令**（`/shell` `/dir` `/restart` 等）默认禁用，需设 `admin_from` 才开；`admin_from = "*"` 等于给所有人 shell 权限，慎用。
- **`mode = "bypassPermissions"`** 让所有工具自动通过，等于把本机交给 Agent，仅在可信单用户环境用。

### 🟠 ilink 网关风险
- ilink 是**第三方**服务，非微信官方，属灰产通道，**有封号风险**。仅自用、别用主力号高频使用。
- 想要更合规的微信通道：改用**企业微信**平台（`type = "wecom"`，走官方 API）。

### 🟡 沙箱隔离的局限
config 里设 `[projects.agent.options.env] CLAUDE_CONFIG_DIR = "<sandbox_dir>"` 可让 Agent 的 settings/skills/plugins 走独立沙箱目录，与全局 `~/.claude` 隔离。**但 ⚠️ 全局用户记忆 `~/.claude/CLAUDE.md` 仍会被读**（Agent 按 HOME 找它，不受 `CLAUDE_CONFIG_DIR` 影响）。要彻底屏蔽全局 CLAUDE.md：把它清空/移到项目级，或用 `run_as_user` 换一个干净 HOME。
> 补充：设了 `CLAUDE_CONFIG_DIR` 沙箱后，启动 cc-connect 时要在**它自己的环境**里也设上同名变量，否则 cc-connect 自身的 skill 解析仍回退到全局 `~/.claude/skills`：
> `CLAUDE_CONFIG_DIR=<sandbox_dir> ./cc-connect`

### 🟡 思考预览（💭）
默认 `[display] thinking_messages = true` 会把模型思考作为 💭 预览发到微信。只想要干净回复就设 `thinking_messages = false`（或 `mode = "quiet"` 连工具噪声一起静音）。改完重启生效。

### 🟡 Mac 睡眠导致不响应
Mac 睡眠会断网 → ilink 长轮询失败 → 不回复（锁屏本身不杀进程）。
- 临时：`caffeinate -di ./cc-connect`
- 永久：`sudo pmset -a displaysleep 0 && sudo pmset -a sleep 0`
> launchd 常驻服务**不**防睡眠，装了 daemon 也得加这步。

### 🔵 进程归属（别让它绑死在某终端）
| 方式 | 命令 | 存活 |
|------|------|------|
| 自己终端跑 | `./cc-connect`（Win: `.\cc-connect.exe`） | 终端关了就停 |
| 常驻服务 | macOS: `./cc-connect daemon install`（launchd）；Windows: `.\cc-connect.exe daemon install`（schtasks+PowerShell） | 开机自启、崩溃自重启 |
- 服务管理：`daemon status` / `daemon logs -f` / `daemon stop` / `daemon uninstall`
- 实例锁冲突加 `--force`。
- macOS daemon 必须在仓库目录执行 `daemon install`（靠 `WorkingDirectory/config.toml` 找配置），且会把安装时 PATH 焊进服务——确保 PATH 含 Agent CLI 所在目录。

### 🔵 多微信号
一个项目下加多个 `[[projects.platforms]] type = "weixin"` 块（各自 token/account_id），逐个扫码：
```bash
./cc-connect weixin setup --project my-wechat --platform-index 1   # 第 1 个号
./cc-connect weixin setup --project my-wechat --platform-index 2   # 第 2 个号
```

### 🔵 Windows 差异速查
配置/weixin 流程/agent 选项与 macOS 完全一致，系统层差异：二进制 `.exe`；常驻服务走 **schtasks+PowerShell**（非 launchd）；默认 shell 是 **powershell**；进程清理用 **`taskkill /T /F`**；数据目录 `%USERPROFILE%\.cc-connect`。`/shell` 跑 PowerShell，其余斜杠命令不变。
